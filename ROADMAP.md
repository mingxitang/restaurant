本轮已完成修复（2026-06-08）：

- 新增 `CurrentUserService`，顾客下单、支付、催单、评价、呼叫服务员统一从 JWT 当前用户取身份。
- `/api/orders/**` 不再开放给顾客角色，顾客只能通过 `/api/customer/**` 访问自己的订单。
- 支付增加 `PENDING` 状态校验和优惠金额上限校验。
- 评价增加订单归属校验和同一订单重复评价拦截。
- Web 端 `token/user` 从 `localStorage` 调整为 `sessionStorage`，登录默认账号改为 Vite 环境变量控制。
- 小程序移除默认测试账号和请求体 `userId`，并优化菜单购物车数量匹配与金额计算。
- 后端列表分页已下沉到 MyBatis 层，`PageUtils` 不再做全量 `List` 的内存分页。
- `migration-add-detail-status.sql` 不再重复创建 `v_kitchen_queue`，最新版视图集中在初始化脚本和当前修复迁移中维护。

本轮复核结论（2026-06-07）：

1. 旧版 `项目CodeReview报告.md` 中关于 JWT 密钥、数据库密码、全局异常泄露、DTO 校验、登录限流、单元测试缺失、Swagger、Docker Compose 等问题，当前代码已完成修复或补齐。
2. 当前更值得优先处理的是“顾客身份与订单归属”这一类访问控制问题，它们比样式拆分或普通重构更接近业务正确性与数据安全。
3. 管理端与顾客端当前都可以正常执行 `npm run build`，后端当前可以通过 `mvn test`，说明项目已具备继续推进权限治理与接口测试的基础。
